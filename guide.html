How to Update the DLBC Website

This page is for whoever is helping run the website — no coding needed. Everything below is done in Google Sheets, the same way you'd use a spreadsheet on your phone or computer.

The short version: Sermons and Activities live in a Google Sheet. Add a row, the website picks it up automatically the next time someone visits. The Daily Devotion needs nothing at all — it already changes on its own every day.

1. How the Daily Devotion works (nothing to do)

The homepage shows a different Bible verse and reflection every day, picked automatically based on today's date. There are 60 devotions built in, cycling roughly every two months, forever, with zero admin work.

  • To add more devotions later, open assets/devotions.js and copy one entry to add a new one. Ask your IT volunteer, or come back to this Claude conversation for help — it only takes a minute.

2. Set up the Google Sheet (one-time setup)

  1. Create a new Google Sheet (from any Google account the church controls — a shared "DLBC Media" account is a good idea).
  2. Rename Tab 1 to Sermons. In row 1, type these column headers exactly: date, title, speaker, passage, youtube, summary.
    • date — format like 2026-07-19 (year-month-day).
    • youtube — paste the full YouTube link of the sermon.
  3. Rename Tab 2 to Activities. In row 1, type: date, title, time, location, description.
  4. Add a row for each sermon or activity, same way you'd fill in any spreadsheet.

3. Publish the sheet so the website can read it

  1. In Google Sheets, go to File → Share → Publish to web.
  2. Under "Link", choose the Sermons tab specifically, and set the format to Comma-separated values (.csv), then click Publish.
  3. Copy the link it gives you — it will look like https://docs.google.com/spreadsheets/d/e/2PACX-.../pub?gid=0&single=true&output=csv.
  4. Repeat the same steps for the Activities tab.

4. Connect the links to the website

Open assets/app.js near the top, and paste your two links in place of the placeholders:

  • sermonsCsvUrl → your Sermons publish link
  • activitiesCsvUrl → your Activities publish link

Save the file and reload the website — your real sermons and activities will now replace the sample content shown by default. If you're not comfortable editing code, share your two published links back in this Claude conversation and this file can be updated for you.

5. Add a new sermon or activity, going forward

  1. Open the Google Sheet.
  2. Add a new row with the details.
  3. That's it — no need to touch the website files again. The site reads the sheet fresh every time someone opens it.
This also gives you the Archive for free: because every sermon ever added stays as a row in the Sermons tab, the site's Archive section automatically lists and lets visitors search all of them — nothing extra to maintain.

6. Putting the website online

This folder is a complete, ready-to-host website. Two free, beginner-friendly options:

  • Netlify Drop — go to app.netlify.com/drop and drag the whole website folder in. It gives you a live link in seconds, and you can add a free custom subdomain.
  • GitHub Pages — upload the folder to a GitHub repository and turn on Pages in the repository settings. Good if a volunteer is already comfortable with GitHub.

Either way, keep this folder structure exactly as it is — index.html, guide.html, and the assets folder together — when you upload.

7. Questions

If anything here is unclear, bring this guide and your question back to this Claude conversation — it has the full context of how this site was built and can walk you through any step, or make the change directly.
